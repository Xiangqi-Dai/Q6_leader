# air_sniffer 空气成分嗅探

Target：对 AIR-MOD-001 七合一空气传感器 + 外接九合一气体传感模组的环境数据进行采集与分析，供后端和前端展示 **CO2 / 甲醛 / VOC / PM2.5 / PM10 / 温湿度 / 氧气 / 硫化氢 / 一氧化碳 / 二氧化硫 / 二氧化氮 / 甲烷（可燃气）/ 氨气 / 磷化氢 / 环氧乙烷 / 空气风险等级**。

参考硬件：

- `AIR-MOD-001` 七合一传感器模组：CO2 / 甲醛 / VOC / PM2.5 / PM10 / 温湿度
- 外接九合一气体传感器：O2 / H2S / CO / SO2 / NO2 / CH4 / NH3 / PH3 / ETO

两类传感器均通过 ESP32 暴露的统一 HTTP 接口读取，例如 `http://192.168.8.89/api/sensor`。

---

## 1. `hardware_sender` 数据采集

- 插件类：`plugins.air_sniffer.plugin.AirSnifferVitalPlugin`
- 采集逻辑：
  - 默认从 `sensor_url` 读取 JSON 数据
  - 当前接口返回示例（七合一 + 九合一字段统一在一个 JSON 中）：
    ```json
    {
      "co2": 550, "hcho": 22, "voc": 0, "pm25": 1, "pm10": 1,
      "temperature": 32.5, "humidity": 33.9,
      "o2": 20.9, "h2s": 1, "co": 5, "so2": 0.2, "no2": 0.1,
      "ch4": 0, "nh3": 2, "ph3": 0.03, "eto": 0.1
    }
    ```
  - 对数值字段做统一转换，缺失或非法值按 `0` 处理
  - 根据阈值生成 `risk_level`、`risk_score`、`main_factor`、`summary`
- **异常处理**：单次读取失败仍上报上一次快照，`status` 为 `error`，`error` 写入简短原因；不中断插件进程。

### 1.1 字段类型约定（int / float）

七合一原有字段沿用既有类型；九种新增气体按量程与阈值分辨率需要选定 `int` / `float`：

| 字段 | 类型 | 单位 | 选型理由 |
| --- | --- | --- | --- |
| `o2` | float | %vol | 典型值 20.9，缺氧警戒线 19.5，必须保留小数 |
| `h2s` | int | ppm | 0-100 量程，报警阈值在整 ppm 量级 |
| `co` | int | ppm | 0-1000 量程 |
| `so2` | float | ppm | 0-20 量程，阈值在个位 ppm 以下 |
| `no2` | float | ppm | 0-20 量程，阈值在个位 ppm 以下 |
| `ch4` | int | %LEL | 0-100，报警阈值 10/25 为整数 |
| `nh3` | int | ppm | 0-100 量程 |
| `ph3` | float | ppm | 剧毒，阈值 0.0x ppm 级，必须小数 |
| `eto` | float | ppm | 阈值 1-2 ppm 级，需小数分辨率 |

> 后续如传感器精度升级或现场标定需要，统一改类型声明 + 解析行即可。

### `config.yaml` 的 `kwargs`

| 参数 | 说明 |
| --- | --- |
| `sensor_url` | 空气传感器 HTTP 地址，如 `http://192.168.8.89/api/sensor` |
| `timeout_sec` | 单次请求超时时间，默认建议 `3.0` 秒 |

### 运行依赖

- AIR-MOD-001 与九合一气体传感器均已上电并稳定输出数据
- 若通过 ESP32 读取，ESP32 需能访问传感器并提供 HTTP JSON 接口
- `hardware_sender` 所在设备需能访问 `sensor_url`
- 插件只依赖 Python 标准库，不需要额外安装 requests

### 建议采集周期

AIR-MOD-001 默认约 1s 上传一次数据；HTTP 轮询时 `interval_sec` 建议 **2s ~ 5s**。

## 3. 风险评估阈值

`_assess_risk` 遍历下表全部指标，取单项风险分最高者作为 `main_factor`，并汇总为 `risk_level` 与 `risk_score`。阈值数值参考下列国标，**具体数值须现场标定后再次确认**。

- 阈值分两条线：`good` 以下 → `normal`；`good`~`bad` 之间 → `warning`；`bad` 以上 → `danger`。
- `risk_score` > 70 判为 `danger`；> 40 判为 `warning`。

### 3.1 线性阈值（浓度越低越好，`_score_linear`）

| key | label | 单位 | 类型 | good | bad | 依据 |
| --- | --- | --- | --- | --- | --- | --- |
| `co2` | CO₂ | ppm | int | 1000 | 2000 | GB/T 18883-2022 |
| `hcho` | 甲醛 | ug/m³ | int | 80 | 100 | GB/T 18883-2022 |
| `voc` | VOC | ug/m³ | int | 200 | 400 | GB/T 18883-2022 |
| `pm25` | PM2.5 | ug/m³ | int | 35 | 75 | GB/T 18883-2022 |
| `pm10` | PM10 | ug/m³ | int | 50 | 150 | GB/T 18883-2022 |
| `h2s` | 硫化氢 | ppm | int | 7 | 15 | GBZ 2.1-2019 MAC ~7 / STEL 15 |
| `co` | 一氧化碳 | ppm | int | 17 | 26 | GBZ 2.1 PC-TWA ~17 / STEL ~26 |
| `so2` | 二氧化硫 | ppm | float | 2 | 5 | GBZ 2.1 PC-TWA ~2 / STEL ~4 |
| `no2` | 二氧化氮 | ppm | float | 3 | 5 | GBZ 2.1 PC-TWA ~3 / STEL ~5 |
| `ch4` | 甲烷(可燃气) | %LEL | int | 10 | 25 | GB 15322 预警 10 / 一级报警 25 |
| `nh3` | 氨气 | ppm | int | 29 | 43 | GBZ 2.1 PC-TWA ~29 / STEL ~43 |
| `ph3` | 磷化氢 | ppm | float | 0.05 | 0.3 | GBZ 2.1 MAC ~0.035（剧毒从严） |
| `eto` | 环氧乙烷 | ppm | float | 1 | 2 | GBZ 2.1 PC-TWA ~1.1 |

### 3.2 区间阈值（双向，过低过高都危险，`_score_range`）

| key | label | 单位 | 类型 | good_range | bad_range | 依据 |
| --- | --- | --- | --- | --- | --- | --- |
| `temperature` | 温度 | ℃ | float | (15, 30) | (5, 38) | 舒适区间 |
| `humidity` | 湿度 | %RH | float | (30, 70) | (20, 80) | 舒适区间 |
| `o2` | 氧气 | %vol | float | (19.5, 23.5) | (18.0, 25.0) | GB 30871 缺氧 / 富氧危险作业 |

## 4. `backend_receiver` 数据处理

- 插件：`plugins.air_sniffer.plugin.AirSnifferBackendPlugin`
- 推荐处理方式：
  - 校验发送端 `vita_data` 必要字段
  - 透传空气指标数值与分析字段（七合一 + 九合一）
  - 将最新快照写入设备状态缓存
  - 当 `risk_level` 为 `warning` 或 `danger` 时，可生成告警或前端提示
  - 通过 `publish_vital_update` 推送到 WebSocket

`backend_receiver/config.yaml` 的 `receiver.plugins.air_sniffer` 需 `enabled: true`。

## 6. `frontend_dashboard` 前端 UI

由于数值指标多达 16 项，且量级差异极大（如 PH3 阈值 0.0x ppm，CO 量程 0-1000 ppm），前端按语义分组展示，避免平铺导致拥挤与趋势图失真。

### 6.1 实时卡片（分组）

- **常规空气**：CO2、甲醛、VOC、PM2.5、PM10、温度、湿度、氧气 O2
- **有毒有害**：CO、H2S、SO2、NO2、甲烷 CH4、NH3、PH3、ETO

每张卡片复用现有 `metric-card` 结构：标签、数值、单位、状态徽标（normal / warning / danger）。

### 6.2 风险摘要

根据 `risk_level` 展示正常 / 关注 / 危险状态，并显示 `risk_score`、`main_factor`、`summary`。

### 6.3 趋势图

- **污染物趋势**：保留原有 CO2 / VOC / PM2.5 / 甲醛 / PM10 轴
- **有毒气体趋势**：新增 CO / H2S / SO2 / NO2 / CH4 / NH3 / PH3 / ETO 轴；因量级差异大，按需提供指标切换或子图
- **温湿度 / 氧气趋势**：温度、湿度、O2（%vol）

### 6.4 告警提示

- `warning`：黄色提示，建议关注或通风
- `danger`：红色提示，建议立即处置并复测
- 有毒气体（H2S / PH3 / ETO 等）触发 `danger` 时，优先级高于常规污染物

### 6.5 气体热力图

`utils/airHeatmap.ts` 的 `AirHeatmapMetric` 与下拉选项需同步新增 9 种气体，并补充各自归一化范围（`AIR_METRIC_RANGES`）与单位。
