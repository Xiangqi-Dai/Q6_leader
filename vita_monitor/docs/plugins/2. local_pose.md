# local_pose 局部定位

表示 具身智能体 在**局部地图** 中的定位位置。

数据来自 ROS2 话题：数据格式与 ROS2 标准位姿数据 `geometry_msgs/msg/PoseStamped` **同构**。


---

## 1. `hardware_sender` 数据采集

- 通过 `config.yaml` 的 `kwargs` 配置：
  - **pose**：`pose_topic`、`pose_message_type`（默认 `geometry_msgs/msg/PoseStamped`）、`pose_field_path`
- **字段路径** `map_field_path` / `pose_field_path`：以 `:` 分隔路径段，首尾 `:` 可忽略；仅 `:` 或空表示根对象即整条消息。
  > e.g. `map_topic=/glio_map`，`map_field_path=:field1:subfield:`  
  > 则 map 数据 = `msg_from_topic("/glio_map").field1.subfield`（须为 PointCloud2）
  > e.g. `pose_topic=/get_pose`，`pose_field_path=:head_pose:`  
  > 则 location 数据 = `msg_from_topic("/get_pose").head_pose`（须为 PoseStamped）


**最小 `location` 载荷**

| 字段 | 说明 |
| --- | --- |
| `frame_id` | Pose header.frame_id |
| `position` | `x, y, z` |
| `orientation` | 四元数 `x, y, z, w` |

运行环境：需在**已 source 的 ROS 2** 下启动 `hardware_sender`；依赖 `rclpy`、`sensor_msgs_py`（ROS2 自带）。位姿序列化见 `hardware_sender/helpers/ros_common/`。

---

## 2. `hardware_sender` → `backend_receiver`（MQTT Envelope 内 `vita_data`）

外层 Envelope 仍遵守 `device_id`、`timestamp`、`vita_type`、`data_mode`、`vita_data`（见 `00_Architecture_整体架构.md`）。

```json
{
  "vita_type": "local_pose",
  "vita_data": {
    "location": {} // ROS2 侧，标准 位置msg数据格式  geometry_msgs/msg/PoseStamped
  }
}
```
---

## 3. `backend_receiver` 数据处理

不做处理，以plugin规范，转发给前端

---

## 4. `backend_receiver` → `frontend_dashboard`

```json
{
  "vita_type": "local_pose",
  "vita_data": {
    "location": {} // ROS2 侧，标准 位置msg数据格式  geometry_msgs/msg/PoseStamped
  }
}
```


---

## 5. `frontend_dashboard` 前端 UI

- 在 **local_map** 专用卡片上绘制：，`location` 为位姿箭头（与点云同一平面假设）。
- 在 **local_map** 专用卡片上绘制 信号强度热力图（前端每接收到一个heartbeat，则找与heartbeat发送时间最近的 local_pose，然后结合二者信息 得到 （信号强度，智能体定位）的pair，根据这个pair更新热力图
- 该体征不在通用 `VitalColumn` 中展示原始 JSON，避免大块数据刷屏。
