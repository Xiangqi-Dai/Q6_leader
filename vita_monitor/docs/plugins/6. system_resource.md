# system_resource 系统计算资源监控

监控**机器人本体**的 **CPU 占用率、CPU 温度、内存使用率、GPU 占用率、GPU 显存使用率、GPU 温度**等计算资源消耗指标，帮助运维人员实时掌握具身智能体的算力负载状态。

| 对比项 | `heartbeat` | `system_resource` |
| --- | --- | --- |
| 监控对象 | 网络连通性（公网 / 局域网） | 本机计算资源（CPU / 内存 / GPU） |
| 业务含义 | 机器人是否能上云 | 机器人算力是否过载 |
| 典型场景 | 网络质量评估、断线告警 | 推理任务过载检测、内存泄漏排查 |
| 采集频率建议 | 2s | **5s**（平衡精度与开销） |

---

## 0. 方案总览

```text
config.yaml (gpu 相关配置)
        │
        ▼
SystemResourceVitalPlugin.collect_vita_data()
        │
        ├── psutil.cpu_percent()          → CPU 占用率
        ├── cpu_temperature.py            → CPU 温度（可选，见 §2.2）
        ├── psutil.virtual_memory()       → 内存使用率
        └── nvidia-smi --query-gpu ...    → GPU 占用率 / 显存 / 温度（可选）
                │
                ▼
vita_data（含 cpu / memory / gpu 结构化指标）
        │
        ├── MQTT → backend_receiver（透传）
        ├── frontend_dashboard（仪表盘 + 时序折线）
        └── local_dashboard Panel（断网时 LAN 内仍可看）
```

**设计原则**

1. **轻量采集**：CPU 使用 `psutil`（1s 阻塞采样），内存使用 `psutil`（非阻塞），CPU 温度优先 `psutil.sensors_temperatures()`、兜底读 `/sys/class/thermal`，GPU 使用 `nvidia-smi`（subprocess），整体开销可控。
2. **GPU 可选兼容**：通过 `shutil.which("nvidia-smi")` 检测 GPU 可用性——无 NVIDIA GPU 的设备自动跳过 GPU 指标，不影响 CPU / 内存数据上报。
3. **CPU 温度可选兼容**：无 lm-sensors / thermal 节点时 `cpu.temperature_available = false`，`cpu.temperature = -1.0`，不影响占用率上报。
4. **失败不阻断**：单个指标采集异常时该字段赋哨兵值，整包仍上报；插件进程不退出。
5. **透传后端**：`backend_receiver` 插件不做数据变换，直接转发原始结构，前端负责展示。

## 2. `hardware_sender` 数据采集

### 2.1 采集方法

| 指标 | 采集方式 | 说明 |
| --- | --- | --- |
| CPU 总占用率 | `psutil.cpu_percent(interval=1.0)` | 0-100%，阻塞 1s 测量（在 5s 间隔下可接受） |
| CPU 每核心占用率 | `psutil.cpu_percent(interval=1.0, percpu=True)` | 返回列表，长度等于核心数 |
| CPU 核心数 | `psutil.cpu_count(logical=True)` | 逻辑核心数，静态值 |
| CPU 温度 | `psutil.sensors_temperatures()` → `/sys/class/thermal/thermal_zone*/temp` | 摄氏度；取 CPU 相关传感器读数的**最大值**作为代表值 |
| CPU 温度可用性 | 采集成功时为 `true` | 无传感器时为 `false`，`temperature = -1.0` |
| 内存总量 | `psutil.virtual_memory().total` | 字节，插件内转为 MB |
| 内存已用 | `psutil.virtual_memory().used` | 字节，插件内转为 MB |
| 内存可用 | `psutil.virtual_memory().available` | 字节，插件内转为 MB |
| 内存使用率 | `psutil.virtual_memory().percent` | 0-100% |
| GPU 使用率 | `nvidia-smi --query-gpu=utilization.gpu` | 0-100% |
| GPU 显存总量 | `nvidia-smi --query-gpu=memory.total` | MiB |
| GPU 显存已用 | `nvidia-smi --query-gpu=memory.used` | MiB |
| GPU 显存使用率 | `memory.used / memory.total * 100` | 0-100%，插件内计算 |
| GPU 温度 | `nvidia-smi --query-gpu=temperature.gpu` | 摄氏度 |
| GPU 设备名 | `nvidia-smi --query-gpu=name` | 如 `NVIDIA Jetson AGX Orin` |

### 2.2 CPU 温度兼容策略

1. **`kwargs.cpu_temperature_enabled`**（默认 `true`）控制是否尝试采集 CPU 温度。
2. **优先**调用 `psutil.sensors_temperatures()`，匹配 `coretemp` / `k10temp` / `cpu` / `acpitz` 等传感器名，取匹配读数中的最大值。
3. **兜底**读取 Linux `/sys/class/thermal/thermal_zone*/`：解析 `type` 与 `temp`（毫摄氏度），过滤 GPU 专属 zone，取 CPU / SoC 相关 zone 的最大值。
4. **均未找到**：`cpu.temperature_available = false`，`cpu.temperature = -1.0`；`status` 仍为 `ok`（占用率正常时）。
5. 采集模块：`hardware_sender/plugins/system_resource/cpu_temperature.py`。

### 2.3 GPU 兼容策略

1. 插件初始化时调用 `shutil.which("nvidia-smi")` 检测可执行文件。
2. **找到**：每轮采集调用 `nvidia-smi --query-gpu=... --format=csv,noheader,nounits`，解析输出。
3. **未找到**：`gpu.available = false`，`gpu.devices = []`，不再尝试 GPU 采集。
4. 运行时 `nvidia-smi` 子进程失败（如驱动异常）：本次 `gpu.devices` 保持上一轮有效数据，`gpu.available` 仍为 `true`，`error` 字段记录异常原因。

### 2.4 `config.yaml` 配置

```yaml
sender:
  plugins:
    system_resource:
      enabled: true
      data_mode: real
      class: plugins.system_resource.plugin:SystemResourceVitalPlugin
      interval_sec: 5.0          # 采集频率：每 5s 一轮
      qos: 0
      kwargs:
        cpu_sample_interval: 1.0   # CPU 百分比采样阻塞时长（秒）
        cpu_temperature_enabled: true  # 是否采集 CPU 温度
        gpu_enabled: true          # 是否尝试 GPU 采集；false 时跳过 nvidia-smi 调用

local_dashboard:
  panel_order:
    - heartbeat
    - ip_peer_monitor
    - system_resource
    # ...
  vitals:
    system_resource:
      push_to_ui: true
      ui_history_limit: 60       # 约 5 分钟 @ 5s 间隔
```

#### `kwargs` 字段说明

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `cpu_sample_interval` | `float` | `1.0` | `psutil.cpu_percent()` 的采样间隔（秒）；值越大越准但越慢，**不超过 `interval_sec` 的 50%** |
| `cpu_temperature_enabled` | `bool` | `true` | 是否采集 CPU 温度；无传感器时可置 `false` |
| `gpu_enabled` | `bool` | `true` | 是否启用 GPU 指标采集；无 GPU 设备可置 `false` 减少一次子进程调用 |

Mock 段可选：

```yaml
mock:
  cpu_temp_range: [38.0, 68.0]
  cpu_temperature_enabled: true
```

### 2.5 运行依赖

- **必须**：Python `psutil` 库（`pip install psutil`）。
- **可选（CPU 温度）**：Linux `thermal`  sysfs，或已安装 lm-sensors 且 `psutil.sensors_temperatures()` 可读。
- **可选**：NVIDIA 驱动 + `nvidia-smi` 命令（GPU 指标需要；无 GPU 时自动降级）。
- 读取 thermal sysfs **无需** root / sudo 权限。

### 3.4 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | `string` | `ok`：采集完成；`error`：全局采集失败 |
| `collected_at` | `float` | 采集完成时的 Unix 时间戳（秒） |
| **`cpu`** | **object** | **CPU 相关指标** |
| `cpu.percent` | `float` | CPU 总占用率（0-100）；失败时 `-1.0` |
| `cpu.per_core` | `list[float]` | 每核心占用率；失败时 `[]` |
| `cpu.core_count` | `int` | 逻辑核心数；失败时 `0` |
| `cpu.temperature` | `float` | CPU 代表温度（℃）；不可用时 `-1.0` |
| `cpu.temperature_available` | `bool` | 是否成功读到 CPU 温度 |
| **`memory`** | **object** | **内存相关指标** |
| `memory.total_mb` | `float` | 内存总量（MB） |
| `memory.used_mb` | `float` | 已用内存（MB） |
| `memory.available_mb` | `float` | 可用内存（MB） |
| `memory.percent` | `float` | 内存使用率（0-100）；失败时 `-1.0` |
| **`gpu`** | **object** | **GPU 相关指标** |
| `gpu.available` | `bool` | 系统是否有可用的 NVIDIA GPU |
| `gpu.devices` | `list[object]` | 逐 GPU 设备指标；无 GPU 时为 `[]` |
| `gpu.devices[].index` | `int` | GPU 索引（多卡场景） |
| `gpu.devices[].name` | `string` | GPU 型号名称 |
| `gpu.devices[].utilization_percent` | `float` | GPU 计算使用率（0-100） |
| `gpu.devices[].memory_total_mb` | `float` | 显存总量（MB） |
| `gpu.devices[].memory_used_mb` | `float` | 显存已用（MB） |
| `gpu.devices[].memory_percent` | `float` | 显存使用率（0-100） |
| `gpu.devices[].temperature` | `float` | GPU 温度（℃） |
| `error` | `string` | 插件级错误描述；成功时为空串 |

## 4. `backend_receiver` 数据处理

- 插件：`plugins.system_resource.plugin.SystemResourceBackendPlugin`
- 策略：**透传**发送端 `vita_data`（与 `ip_peer_monitor` 相同），`process_vita_data` 直接返回 sender 数据。
- 历史：默认 `persist_vital_history: true`，写入 ClickHouse 供前端时序查询。

`backend_receiver/config.yaml` 示例：

```yaml
receiver:
  plugins:
    system_resource:
      enabled: true
      class: "plugins.system_resource.plugin:SystemResourceBackendPlugin"
      kwargs:
        persist_vital_history: true
        history_retention_hours: 24
```

## 10. 联调检查清单

- [ ] `hardware_sender` / `backend_receiver` 均启用 `system_resource`，`vita_type` 一致。
- [ ] `psutil` 已安装在硬件端 Python 环境中。
- [ ] 有 thermal / lm-sensors 的设备：`cpu.temperature_available = true` 且温度合理。
- [ ] 无 CPU 温度传感器的设备：`cpu.temperature_available = false`，占用率仍正常上报。
- [ ] 有 NVIDIA GPU 的设备：`nvidia-smi` 可执行，GPU 指标正常上报。
- [ ] 无 NVIDIA GPU 的设备：`gpu.available = false`，CPU / 内存指标正常上报。
- [ ] `interval_sec: 5.0`，单轮采集耗时 < 3s（含 CPU 1s 阻塞采样 + nvidia-smi 子进程）。
- [ ] WebSocket 收到 `vita_type=system_resource` 的 `vital.updated`。
- [ ] 公网断联 + Local Dashboard 启用时，内网页仍刷新资源面板。
