# local_map 局部地图

表示 具身智能体 正在使用的 **局部地图（本地地图）** （雷达点云图）。

数据来自 ROS 2 话题：数据格式 与 `sensor_msgs/msg/PointCloud2` **同构**（使用雷达点云图表示 local map）。

---

## 1. `hardware_sender` 数据采集

### 1.1 配置（`config.yaml` → `kwargs`）

| 参数 | 说明 |
| --- | --- |
| `source` | `ros`（ROS 2 话题）或 `xos`（XOS HTTP） |
| `map_topic` | ROS 点云话题名（`source: ros`） |
| `map_message_type` | 默认 `sensor_msgs/msg/PointCloud2` |
| `map_field_path` | 以 `:` 分隔的字段路径，首尾 `:` 可忽略；仅 `:` 或空表示根对象 |
| `xos_host` | XOS 服务地址（`source: xos`，默认 `http://localhost:1888`） |

**字段路径示例**

> `map_topic=/glio_map`，`map_field_path=:field1:subfield:`  
> 则 map 数据 = `msg_from_topic("/glio_map").field1.subfield`（须为 PointCloud2）

运行环境：需在**已 source 的 ROS 2** 下启动 `hardware_sender`（`source: ros`）；依赖 `rclpy`、`sensor_msgs_py`。ROS 序列化与字段路径见 `hardware_sender/helpers/ros_common/`。

### 1.2 点云载荷（完整、不抽稀）

采集端**不做** `map_max_points` / `map_stride` 等抽稀或截断；从源消息解析出的全部 `(x, y)` 点均保留在 `points` 中。

| 字段 | 说明 |
| --- | --- |
| `frame_id` | 点云 `header.frame_id` |
| `points` | `[[x, y], ...]` 完整 2D 点列表（用于底图） |
| `point_count` | `len(points)` |
| `width` / `height` | 原消息 `width` / `height` |

> 历史字段 `sampled_count`、`stride` 及配置项 `map_max_points`、`map_stride` 已废弃，不再产出。

### 1.3 采集频率与发送策略（内容比对去重）

- **采集频率**：仍由插件 `interval_sec` 控制；每个周期执行一次 `collect_*_vita_data()`。
- **发送频率**：与采集频率**解耦**。每次采集后，对 `vita_data` 做**内容指纹**比对（剔除 `error` 等瞬时字段；`map` 内不含采集时间戳）。
- **若内容与上次成功入池的数据一致**：跳过 `put_data_to_pool`，不向 `info_pool` / MQTT / `local_dashboard` 二次推送。
- **若内容发生变化**：正常入池并发送。

实现位置：`hardware_sender/plugins/local_map/plugin.py`（覆写采集循环）；指纹工具见 `hardware_sender/plugins/local_map/content_fingerprint.py`。

## 2. `hardware_sender` → `backend_receiver`（MQTT Envelope 内 `vita_data`）

外层 Envelope 仍遵守 `device_id`、`timestamp`、`vita_type`、`data_mode`、`vita_data`（见 `00_Architecture_整体架构.md`）。

```json
{
  "vita_type": "local_map",
  "vita_data": {
    "map": {
      "frame_id": "map",
      "points": [[0.1, 0.2]],
      "point_count": 1,
      "width": 1,
      "height": 1
    }
  }
}
```

---

## 3. `backend_receiver` 数据处理

不做处理，以 plugin 规范，转发给前端。

---

## 4. `backend_receiver` → `frontend_dashboard`

```json
{
  "vita_type": "local_map",
  "vita_data": {
    "map": {
      "frame_id": "map",
      "points": [[0.1, 0.2]],
      "point_count": 1,
      "width": 1,
      "height": 1
    }
  }
}
```

---

## 5. `frontend_dashboard` 前端 UI

- 设备详情页提供 **local_map** 专用卡片：Canvas 绘制 `map.points` 为底图散点。
- 点数展示优先读 `point_count`，兼容历史字段 `sampled_count`。
- 该体征不在通用 `VitalColumn` 中展示原始 JSON，避免大块数据刷屏。
