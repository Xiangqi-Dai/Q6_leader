# status 在线/离线监控

表示具身智能体与 **MQTT Broker** 的会话可达性。

### `hardware_sender` 数据采集

直接通过broker/MQTT协议发送，不经过硬件端处理。

### `hardware_sender->backend_receiver` 传输阶段的数据结构：

```json
  "vita_type": "status",
  "vita_data": {
    "online": true,
    "status": "online",
    "reason": "boot",
  }
```

下线/遗嘱（字段以实际发布为准；遗嘱可能缺 `reported_at` / `status`）：

```json
  "vita_type": "status",
  "vita_data": {
    "online": false,
    "status": "offline",
    "reason": "network_lost"
  }
```

### `backend_receiver`数据处理

`StatusBackendPlugin`（`backend_receiver/plugins/status/plugin.py`）对 `vita_data` **透传**。

### `backend_receiver->frontend_dashboard` 传输阶段的数据结构：

如果 对方具身智能体 没有 connect 到我们的broker，就传递 “offline” ，如果连接就传递 “online”

```json
  "vita_type": "status",
  "vita_data": {
    "online": true,
    "status": "online",
    "reason": "boot"
  }
```

### `frontend_dashboard`前端UI

- **机队总览（`HomeView`）**：`isDeviceOnline`（`src/utils/online.ts`）读 `latest.status`——对象优先 `online` 布尔，否则 `status` 字符串；兼容顶层字符串。卡片描边与 `NTag`：**绿=在线 / 红=离线 / 灰=未知**（`success` / `error` / `default`）。进入页 `deviceStore.loadList()`，WebSocket `applyEnvelope` 实时刷新列表中的 `latest`。
- **设备详情（`DeviceDetailView`）**：`VitalColumn` 展示 `detail.latest.status` 键值行；与其它体征相同经 WebSocket `vital.updated` 写入 `latest`。圆点由 `trafficForStatusField` / `trafficForVitalField`：**在线为红（`bad`）、离线为绿（`good`）**——与总览配色语义相反，为产品约定；`reason` 非空为黄（`warn`），空为灰。
- **无 status**：`isDeviceOnline` 为 `null`，总览显示「未知」。
